# `run_sim.py` 启动参数说明

在 `sim` 目录下执行；`run_sim.py` 会 **`bazel run //cpp:sim_runner`** 调用 C++ 闭环仿真。Python 规划与旧版入口见仓库 **`pysim/`**（含 `run_default.sh`）。

也可使用 `python3 run_sim.py --help` 查看 argparse 自带的帮助（与下表一致）。

---

## 必参与场景

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--scenario-dir` | 路径 | （必填） | 场景目录，需包含 `scenario_meta`、`dynamic_objects`、`lane_graph` 等 JSON（由仓库 `tools` 转换流程产出）。若缺少 `scenario_meta` 中的 init/goal，程序会退出并报错。 |

---

## 规划器与路由

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--planner` | 字符串 | `local_dwa` | C++ 已注册名称：`reference_tracker`、`goal_seek`、`local_dwa` 等（见 `sim_runner` 启动失败时 stderr 列出的列表）。 |
| `--reference-source` | 枚举 | `map` | `map`：车道图 BFS 参考线；`sdc`：Waymo 自车轨迹作 reference。 |
| `--desired-speed` | 浮点 | `13.9` | 期望速度（m/s），供规划器参考。 |
| `--reference-step` | 浮点 | `1.0` | 地图路由参考线重采样步长（米），`map` 模式生效。 |

---

## 仿真步长与时长

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--dt` | 浮点 | `0.1` | 仿真 / 控制时间步（秒）。 |
| `--max-seconds` | 浮点 | `0.0` | 大于 0 时覆盖场景时长，只跑前若干秒；`0` 表示用场景完整时间范围。 |

---

## 自车几何与动力学

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--ego-length` | 浮点 | `4.5` | 车长（米）。 |
| `--ego-width` | 浮点 | `1.85` | 车宽（米）。 |
| `--ego-wheelbase` | 浮点 | `2.7` | 轴距（米）。 |
| `--ego-rear-overhang` | 浮点 | `0.95` | 后悬（米）。 |
| `--ego-max-speed` | 浮点 | `33.3` | 自车最大速度上限（m/s）。 |

---

## 输出与 SimLog

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--output` | 路径 | `<仓库根>/output/log/sim_log.json` | 供 grading 离线批处理的 SimLog JSON（`MetricFrameInput`）；与下方 spdlog 文本日志无关。 |
| `--source-tag` | 字符串 | `waymo_sim_cpp` | 写入 SimLog 顶层的 `source` 字段。 |

---

## Sim 调试日志（spdlog，仅仿真侧）

由 `sim/cpp/sim_logger` 写入 `sim_YYYYMMDD_HHMMSS.log`，**只记录** `hyw_sim.proto`，**不包含** grading 的 `MetricFrameInput`。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--log-dir` | 路径 | 空 | 非空时启用 spdlog 文件日志。 |
| `--log-level` | 枚举 | `info` | `trace` / `debug` / `info` / `warn` / `error` / `off`。`debug` 时每帧输出下列 tag 的全量 JSON。 |

**`--log-level debug` 时的 tag（sim 仅 4 类）**

| tag | 含义 |
|-----|------|
| `planner_inputs` | 启动一次：`PlannerInputs` |
| `planner_observation` | 每步：进 planner 前的 `PlannerObservation` |
| `plan_command` | 每步：`PlanCommand` |
| `frame_record` | 每步：积分后 `FrameRecord`（含 `planned_trajectory`） |

另有 `info` 级别的 `run_start`、`simulation_done`。

```bash
tail -f ../output/log/sim_*.log
grep planner_observation ../output/log/sim_*.log | head
```

---

## C++ 评分器 `grading_main`

仅在设置了 `--grading-bin` 且 `--cpp-mode` 不是 `off` 时启用。评分走 pipe 或 `--output` SimLog，**不写入** `sim_*.log`。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--grading-bin` | 路径 | 空 | `grading_main` 可执行文件路径。 |
| `--grading-report` | 路径 | 空 | 默认 `<仓库根>/output/report/grading_report.json`。 |
| `--metrics-config` | 路径 | 空 | metrics JSON。 |
| `--cpp-mode` | 枚举 | `online` | `online` / `offline` / `both` / `off`。 |

---

## 示例

仅仿真 + sim debug 日志：

```bash
cd sim
python3 run_sim.py \
  --scenario-dir ../scenarios/waymo_scenario_5 \
  --log-dir ../output/log \
  --log-level debug \
  --cpp-mode off
```

仿真 + SimLog + 在线评分（spdlog 仍只记 sim 侧）：

```bash
python3 run_sim.py \
  --scenario-dir ../scenarios/waymo_scenario_5 \
  --grading-bin ../grading_mini/bazel-bin/src/entry/grading_main \
  --log-dir ../output/log \
  --log-level debug
```

---

参数实现：`sim/run_sim.py` → `sim/cpp/main.cc`。
